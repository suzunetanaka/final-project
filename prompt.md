# レシピ提案

以下の条件をすべて満たす家庭料理レシピを ${count} 件提案してください。
条件を満たさないレシピは提案しないでください。

## 条件

- 使いたい材料: ${ingredients}
- 食べたいもの・気分: ${craving}
- 調理時間: ${time}
- 難易度: ${difficulty}
- 人数: ${servings}人分

## 優先順位

1. 使いたい材料
2. 食べたいもの・気分
3. 調理時間
4. 難易度

## ルール

- 材料が指定されている場合は、必ずその材料を使用すること。
- 材料が3種類以下の場合は、指定された材料をすべて使用すること。
- 材料が4種類以上の場合は、できるだけ多く使用し、最低でも半分以上使用すること。
- 指定された材料と関係のない料理は提案しないこと。
- 「食べたいもの」に料理名（例：カレー、オムライス、パスタ）が指定されている場合は、その料理またはそのアレンジのみ提案すること。
- 「食べたいもの」が「あっさり」「こってり」「ピリ辛」など味や気分の場合は、料理名・概要・味付け・手順のいずれかに必ず反映すること。
- 「食べたいもの」に料理名と味・気分の両方が含まれる場合は、料理名を優先し、その料理に味・気分を反映すること。
- 調理時間が「こだわらない」以外の場合は、指定された調理時間を超えないこと。
- 難易度が「かんたん」の場合は、工程が少なく初心者でも作れる内容にすること。
- 一般家庭で作れる料理のみ提案すること。
- スーパーで購入できる一般的な材料のみ使用すること。
- 存在しない料理名や現実的でない組み合わせは提案しないこと。

## 出力ルール

- JSON以外の文字は一切出力しないこと。
- Markdownや説明文は不要。
- 出力は必ず「{」で始まり「}」で終えること。
- `recipes` には必ず ${count} 件のレシピオブジェクトを入れること。
- 以下のJSON形式を厳密に守ること。

## JSON形式

```json
{
  "recipes": [
    {
      "title": "レシピ名",
      "category": "料理カテゴリ",
      "time": "20分",
      "minutes": 20,
      "difficulty": "かんたん",
      "difficultyLevel": 1,
      "servings": "${servings}人分",
      "ingredients": [
        "材料 量"
      ],
      "tags": [
        "quick"
      ],
      "summary": "短い説明",
      "steps": [
        "手順1",
        "手順2"
      ],
      "adjustmentTips": "味付け調整の短い助言"
    }
  ]
}
```
